## 1. 项目简介

本项目是一个针对 **Binance BTCUSDT 永续合约** 设计的自动化交易机器人。它基于**顺势回踩**的核心交易思想，在判断出主趋势方向后，寻找关键的支撑/压力区域，并在价格回踩至这些区域时，结合多维度市场信息进行综合判断，最终确认入场信号。

相较于传统策略，本项目引入了三大核心增强模块，旨在提高信号质量、优化风险管理，并提供专业级的策略评估工具：

1.  **市场共识强度评分模块 (Consensus Strength Score)**：在潜在入场点，通过 7 个维度对市场状态进行量化打分，只有评分超过预设阈值的“高共识”机会才会被考虑，有效过滤大量低质量信号。
2.  **结构化退出机制 (Structured Exit Mechanism)**：摒弃了固定的百分比止盈止损，采用“结构化止损 + 分批止盈 + 跟踪止盈”的动态退出策略，让利润奔跑，同时保护本金。
3.  **精细化风险与仓位管理 (Advanced Risk Management)**：实现了基于账户风险百分比的动态仓位计算，并与交易所的最小名义开仓金额（10 USDT）限制进行了联动处理，确保在任何市场波动下都能保持一致的风险暴露。

整个系统采用模块化架构，代码清晰、易于扩展，并配备了完整的**对比回测框架**和**结构化日志系统**，支持实盘交易、测试网交易和模拟运行模式。

| **精细化风控** | 动态计算仓位，并智能处理交易所最小名义金额限制，在风险控制和执行可行性之间取得平衡。 |
| **模块化架构** | 代码按功能高度解耦，分为配置、数据、信号、执行、风控、退出、日志等模块，易于维护和二次开发。 |
| **对比回测** | 内置回测框架，可一键对比“有/无共识评分”两种模式下的策略表现，直观评估模块效果。 |

## 4. 安装与配置

#### 步骤 1: 环境要求

- Python 3.10 或更高版本
- `pip` 包管理器

#### 步骤 2: 下载项目代码

将 `btc_trader` 文件夹完整下载到您的本地或服务器环境中。

#### 步骤 3: 安装依赖

在项目根目录（`btc_trader/`）下打开终端，执行以下命令安装所有必需的 Python 库：

```bash
pip install -r requirements.txt
```

#### 步骤 4: 配置 API Keys (实盘交易)

如果您要进行实盘或测试网交易，需要将您的 Binance API Key 和 Secret Key 设置为环境变量。**请勿将 Keys 硬编码在代码中。**

在终端中执行：

```bash
export BINANCE_API_KEY="your_api_key_here"
export BINANCE_API_SECRET="your_api_secret_here"
```

**提示**：为了安全，建议为该机器人单独创建一个 API Key，并仅授予“允许合约交易”权限，同时绑定 IP 地址。

## 5. 策略详解

#### 5.1 完整交易流程

策略的决策流程严格遵循以下步骤，构成一个完整的交易漏斗：

1.  **1H 趋势过滤**: 使用 EMA(50) 和 EMA(200) 判断 1 小时级别是否处于明确的上升趋势。只有当 `价格 > EMA(200)` 且 `EMA(50) > EMA(200)` 时，才允许寻找做多机会。
2.  **15M 支撑区识别**: 在 15 分钟级别上，利用 `support_zone.py` 模块自动识别由前期高/低点形成的潜在支撑区域。
3.  **市场共识强度评分**: 当价格进入或靠近某个支撑区时，启动 `consensus_score.py` 模块，从 7 个维度进行打分。
4.  **评分阈值过滤**: 只有当总分超过 `config.py` 中设定的阈值时，才认为这是一个高质量的、值得关注的交易机会。
5.  **5M 入场确认**: 在 5 分钟级别上，等待一个明确的看涨 K 线信号（如阳线、锤子线等）作为最终的入场触发条件。
6.  **风险与仓位计算**: 触发入场后，`risk_manager.py` 会根据当前账户净值和风险设置计算开仓数量。
7.  **执行与退出管理**: `main.py` 执行下单，并由 `exit_manager.py` 接管后续的止损、止盈和持仓管理。

#### 5.2 市场共识强度评分模块

这是本策略的核心优势。它将交易员的主观“盘感”转化为可量化、可回测的客观指标。评分维度如下：

| 维度 | 考察内容 | 权重 |
| :--- | :--- | :--- |
| **结构显著性** | 支撑区的形成强度、是否为关键高低点。 | 25% |
| **测试次数质量** | 区域被测试的次数和每次测试的反应。 | 20% |
| **多周期共振** | 该区域在更高时间周期（如 1H、4H）是否同样存在。 | 15% |
| **整数/心理关口** | 区域是否与重要的整数价格（如 65000）重合。 | 10% |
| **技术重合度** | 是否有其他技术指标（如斐波那契回撤位、趋势线）在此汇集。 | 10% |
| **成交量结构** | 区域形成和测试时的成交量分布情况。 | 10% |
| **近期新鲜度** | 区域是近期形成的还是历史久远的。 | 10% |

#### 5.3 结构化退出机制

- **止损逻辑**: 优先使用**结构化止损**。默认止损点设置在 `15M 支撑区下沿` 或 `5M 确认K线低点` 的下方，并增加一个基于 ATR（平均真实波幅）的动态缓冲，以适应不同波动率的市场。同时设有最大/最小止损距离保护，避免风险过大或过小。
- **分批止盈**: 
  - **TP1**: 当盈利达到 1.2R (R = 初始风险单位) 时，自动平仓 35% 并将止损移至成本价（保本）。
  - **TP2**: 当价格接近下一个 15M 压力区时，再平仓 35%。
- **跟踪止盈**: 剩余 30% 的仓位将采用基于 5M 周期 EMA 或结构低点的**跟踪止损**，让利润随趋势奔跑，直至趋势反转信号出现。

## 6. 使用指南

#### 6.1 参数配置 (`config.py`)

所有策略的核心参数都集中在 `config.py` 文件中，方便统一调整和优化。主要可调参数包括：

- **共识评分阈值**: `consensus_score["threshold"]["default"]`
- **各评分维度权重**: `consensus_score["dimensions"]`
- **风险设置**: `risk_management["risk_per_trade_pct"]`
- **止损模式与参数**: `stop_loss`
- **止盈规则**: `take_profit`

#### 6.2 回测 (`backtest.py`)

回测是策略优化的关键。本框架提供了强大的对比回测功能。

1.  **准备数据**: 
    - 创建 `data` 文件夹。
    - 从交易所下载 BTCUSDT 永续合约的 15 分钟 K线历史数据，保存为 CSV 文件（如 `BTCUSDT_15m.csv`）。
    - CSV 文件必须包含以下列（不区分大小写）：`timestamp`, `open`, `high`, `low`, `close`, `volume`。

2.  **运行回测**: 
    在终端执行以下命令，即可运行“有/无共识评分”的对比回测：

    ```bash
    python backtest.py --data data/BTCUSDT_15m.csv
    ```

3.  **解读报告**: 
    程序将输出一份详细的对比报告，直观展示共识评分模块对策略胜率、盈亏比、最大回撤等关键指标的影响。

#### 6.3 实盘交易 (`main.py`)

**警告：实盘交易涉及真实资金风险，请务必在充分理解策略、完成足够回测并小资金测试后谨慎使用。**

- **启动实盘交易**:
  ```bash
  python main.py
  ```

- **使用测试网交易**:
  ```bash
  python main.py --testnet
  ```

- **模拟运行 (Dry Run)**:
  此模式下，机器人会执行所有流程，包括信号生成和日志记录，但不会发出任何真实订单。非常适合在上线前观察机器人的实时行为。
  ```bash
  python main.py --dry-run
  ```

## 7. 日志系统

机器人的所有行为都被详细记录在 `logs/btc_trader.log` 文件中。

- **普通日志**: 记录程序启动、API连接、数据拉取等常规信息。
- **结构化日志**: 当有重大决策时，会输出格式化的报告，例如：
    - **共识强度评分报告**: 详细展示每个维度的得分和最终决策。
    - **入场/拒绝信号报告**: 记录触发的信号参数或被拒绝的原因。
    - **退出事件报告**: 记录止损、止盈等平仓事件的详细信息。

通过查阅日志，您可以清晰地复盘机器人的每一次决策过程。
